# PYGEMC Release 0.4.0

This version includes:

- PyVista variation filtering with `--pyvista-variation`
- Batched PyVista rendering for large geometries with `--pyvista-fast`
- p, theta, and phi residual plots with per-particle CLI filtering

<br/>

# Release notes

## Analyzer

- GEMC's `save_original_track` option saves the original track ID (`otid`), original particle ID (`opid`),
  and original momentum (`opx`, `opy`, `opz`) in true-information output. This provides the
  original-track reference needed for residual plots.
- The analyzer now derives and plots the p, theta, and phi residuals as `delta_p`, `delta_theta`, and
  wrapped `delta_phi` for hits whose `pid` matches `opid`. The residual is the value at the hit minus the
  original-track value.
- Added per-particle plotting through the Python API `pid` argument and the CLI `--pid <PDG ID>` option,
  for example `gemc-analyzer b2_t0_true_info.csv delta_p --data true_info --pid 2212`. Empty PID
  selections display a concise message and exit normally without a Python traceback.

## API

- `-pv`, `-pvb`, and `-pvvtk` render only the first published variation by default, and
  `--pyvista-variation NAME` selects a specific variation without changing SQLite or ASCII output.
- `GConfiguration.show()` prints a variation/run table instead of a compressed `(Variations, Run)` tuple
  when more than one variation is used.
- The variation/run table now records each variation's run at publish time instead of repeating the final
  run.
- `GConfiguration.show()` now prints the variation selected for PyVista rendering when `-pv`, `-pvb`, or
  `-pvvtk` is used.
- Added `--pyvista-fast`, `--no-pyvista-fast`, and `--pyvista-fast-threshold N`. PyVista now defers actor
  creation until display/export time and automatically batches large scenes above the threshold.

## Detailed list of changes and fixes

- The `Publish PyPI` workflow can now be triggered by a published GitHub release and uses that release's
  tag to build and publish the package. Stable releases always publish to PyPI, while pre-releases are
  skipped; the existing manual `workflow_dispatch` with the TestPyPI option is unchanged.
- `gemc-sqlite` now binds user-supplied filter values as SQLite parameters and validates `-what` column
  names against the selected table schema before building a query.
