# PYGEMC Release 0.3.0

This version includes:

- PyVista variation filtering with `--pyvista-variation`
- Batched PyVista rendering for large geometries with `--pyvista-fast`

## API

- `-pv`, `-pvb`, and `-pvvtk` render only the first published variation by
  default, and `--pyvista-variation NAME` selects a specific variation without changing sqlite/ascii output.
- `GConfiguration.show()` prints a variation/run table instead of a compressed `(Variations, Run)` tuple when
  more than one variation is used.
- The variation/run table now records each variation's run at publish time instead of repeating the final run.
- `GConfiguration.show()` now prints the variation selected for PyVista rendering when `-pv`, `-pvb`, or
  `-pvvtk` is used.
- Added `--pyvista-fast`, `--no-pyvista-fast`, and `--pyvista-fast-threshold N`. PyVista now defers actor
  creation until display/export time and automatically batches large scenes above the threshold.

## Detailed list of changes and fixes

- The `Publish PyPI` workflow can now be triggered by a published GitHub release and uses that release's tag to
  build and publish the package. Stable releases always publish to PyPI, while pre-releases are skipped; the
  existing manual `workflow_dispatch` (with the TestPyPI option) is unchanged.
- `gemc-sqlite` now binds user-supplied filter values as SQLite parameters and validates `-what` column names
  against the selected table schema before building the query.
