## Addressed issues

- [Issue #2](https://github.com/gemc/pygemc/issues/2): [High] add_rotation() emits a multi-token rotation string the C++ parser silently drops
- [Issue #3](https://github.com/gemc/pygemc/issues/3): [High] set_rotation() then add_rotation() raises AttributeError due to inconsistent rotations type
- [Issue #4](https://github.com/gemc/pygemc/issues/4): [High] make_polycone() length check uses `and` instead of `or`, missing single-array mismatches
- [Issue #5](https://github.com/gemc/pygemc/issues/5): [High] make_trap() passes an extra `self` to already-bound methods, raising TypeError
- [Issue #6](https://github.com/gemc/pygemc/issues/6): [Medium] GConfiguration ignores constructor args; argparse defaults clobber factory/variation/runno
- [Issue #7](https://github.com/gemc/pygemc/issues/7): [Medium] gemc-sqlite CLI builds queries via raw f-strings (breakage + SQL injection)
- [Issue #8](https://github.com/gemc/pygemc/issues/8): [Low] convert_angle(out='gon') returns radians instead of gradians
- [Issue #15](https://github.com/gemc/pygemc/issues/15): Simplify UI
- [Issue #16](https://github.com/gemc/pygemc/issues/16): Plot smeared quantities

## Detailed list of changes and fixes

- The `Publish PyPI` workflow can now be triggered by a published GitHub release and uses that release's tag to
  build and publish the package. Stable releases always publish to PyPI, while pre-releases are skipped; the
  existing manual `workflow_dispatch` (with the TestPyPI option) is unchanged.
