

## Analyzer

## Geometry

- Added `GVolume.distribute_on_circle(n, radius, phistart=0, phispan=360, align=False, axis='z',
  lunit='mm', aunit='deg')` to replicate a volume at `n` equal angular steps along a circle of the
  given `radius`. Returns a `list[GVolume]`, one copy per step, each named `<name>_i`.
  - Positions each copy at `φᵢ = phistart + i × phispan / n` along the chosen axis (`'x'`, `'y'`, or `'z'`).
  - With `align=True`, appends the angular step φᵢ as a second xyz rotation using GEMC's `doubleRotation:`
    format, so the volume stays radially aligned at every position.
- Added `g4placement_type` attribute to `GVolume` with two modes:
  - `'active'` (default) uses `G4Transform3D`, which rotates and translates the solid. This is the
    Geant4 standard.
  - `'passive'` uses the frame-rotation `G4PVPlacement` constructor, matching the GEMC2/clas12Tags
    convention.
- The selected mode is stored in the SQLite database and drives the Geant4 placement constructor used
  during simulation.

## PyVista

- Changed default PyVista background color from `"white"` to `"#00122B"` (RGB 0, 18, 43 — dark navy).
  The previous white default can be restored with `-pvbg white`.
- Added `-pvbg`, `--pyvista-background-color` to set the base PyVista background color.
- Added `-pvbgt`, `--pyvista-background-top` to set the optional top gradient color.
- Use `-pvbgt none` for a flat background.
- Added `doubleRotation:` branch to `parse_rotation_string()` so that volumes placed with
  `distribute_on_circle(align=True)` render with the correct per-copy orientation in PyVista.
- Added `G4Trap` (general trapezoid) visualization support.
- Added `G4Sphere` (spherical shell section) visualization support.
- Added `G4Polycone` visualization support.
- Added `parse_rotation_string()` to interpret GEMC rotation strings in simple xyz, ordered-axis,
  and compound (`add_rotation`) forms.
- PyVista placement now respects `g4placement_type`: volumes with `'passive'` mode apply the transpose
  of the rotation matrix so that world-frame positions are rendered correctly.
- Aligned ASCII output entries in `gvolume.py` and `gmaterial.py` for improved readability.

## Examples

- Added `scintillator_barrel`: 48 trapezoidal `G4Trap` paddles distributed in a full ring using
  `distribute_on_circle(align=True, axis='z')`. Demonstrates the chord-sizing formula that makes
  adjacent outer faces contiguous for any paddle count, and the `flux` digitization with per-paddle
  identifiers.
