pygemc v0.3.0 expands analyzer plotting and documents the Cherenkov demonstration workflow built around the new y-vs-x plot mode.

# Release notes

This release focuses on analyzer usability for optical-photon studies and adds test coverage for the new plotting path. It also documents the corresponding Cherenkov example updates in GEMC.

## Analyzer

- Added `plot_y_vs_x()` for 2D hit-position maps.
- Added `gemc-analyzer --plot yvsx` for command-line y-vs-x plots.
- Added `--x` and `--y` options to select coordinate columns. Defaults are `avgx` and `avgy`.
- Added `--ylim LOW HIGH` to complement the existing `--xlim LOW HIGH`.
- Added `--position-unit {mm,cm}` for y-vs-x plots. The default is `cm`.
- Updated y-vs-x labels for average hit positions.

## Documentation

- Documented `gemc-analyzer --plot yvsx` usage in the README.
- Added an example command with `--xlim -20 20 --ylim -20 20`.
- Updated the Cherenkov example documentation to describe neutral radiator names and demonstration optical constants.
- Updated the Cherenkov example documentation to use one generated electron for the quick analyzer workflow.

## Tests

- Added analyzer unit coverage for y-vs-x axis limits and centimeter labels.
- Added analyzer CLI coverage that writes a tiny true-info CSV, runs `--plot yvsx --save`, and verifies that a PNG is produced.

## GEMC example integration

- The GEMC Cherenkov example now uses neutral demonstration radiator names:
  - `lowIndexRadiator`
  - `mediumIndexRadiator`
  - `highIndexRadiator`
- The example homepage assets were regenerated with the analyzer y-vs-x mode and `-20 cm` to `20 cm` x/y limits.
- The old gas-named image outputs were replaced by radiator-named files.

# Supported platforms

- Linux
- macOS

# Dependencies

- Python 3.10 or higher
- numpy, pandas, matplotlib
- uproot and awkward for optional ROOT-file analysis via the `root` extra

# Compatibility notes

- Existing histogram usage remains the default analyzer behavior.
- The new y-vs-x CLI mode does not require a positional variable argument.
