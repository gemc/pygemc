pygemc v0.3.0 expands analyzer plotting and documents the Cherenkov demonstration workflow built around the new y-vs-x plot mode.

# Release notes

This release focuses on analyzer usability for optical-photon studies and adds test coverage for 
the new plotting path. It also documents the corresponding Cherenkov example updates in GEMC.

<br/>

## Analyzer

- Added `plot_y_vs_x()` for 2D hit-position maps.
- Added `gemc-analyzer --plot yvsx` for command-line y-vs-x plots.
- Added `--x` and `--y` options to select coordinate columns. Defaults are `avgx` and `avgy`.
- Added `--ylim LOW HIGH` to complement the existing `--xlim LOW HIGH`.
- Added `--position-unit {mm,cm}` for y-vs-x plots. The default is `cm`.
- Updated y-vs-x labels for average hit positions.

<br/>

## Documentation

- Documented `gemc-analyzer --plot yvsx` usage in the README.
- Added an example command with `--xlim -20 20 --ylim -20 20`.
- Documented PyVista background controls for light-background VTK.js documentation exports.
- Updated the Cherenkov example documentation to describe neutral radiator names and demonstration optical constants.
- Updated the Cherenkov example documentation to use one generated electron for the quick analyzer workflow.
- Added `gemc-sqlite` to the README features list and added a full `### gemc-sqlite` section under Command-Line Tools.
- Documented environment activation (`source <prefix>/python_env/bin/activate`) in the Installed with GEMC section.
- Clarified that `/opt/homebrew/bin/python3` should be used when creating virtual environments on macOS with Homebrew.

<br/>

## Tests

- Added analyzer unit coverage for y-vs-x axis limits and centimeter labels.
- Added analyzer CLI coverage that writes a tiny true-info CSV, runs `--plot yvsx --save`, and verifies that a PNG is produced.

<br/>


## GVolume

- Added `g4placement_type` attribute to `GVolume` with two modes:
  - `'active'` (default) — uses `G4Transform3D`, which rotates and translates the solid. This is the Geant4 standard.
  - `'passive'` — uses the frame-rotation `G4PVPlacement` constructor, matching the GEMC2/clas12Tags convention.
- The selected mode is stored in the SQLite database and drives the Geant4 placement constructor used during simulation.

<br/>

## PyVista

- Added `-pvbg`, `--pyvista-background-color` to set the base PyVista background color.
- Added `-pvbgt`, `--pyvista-background-top` to set the optional top gradient color.
- Use `-pvbgt none` for a flat background.
- Added `G4Trap` (general trapezoid) visualization support.
- Added `G4Sphere` (spherical shell section) visualization support.
- Added `G4Polycone` visualization support.
- Added `parse_rotation_string()` to interpret GEMC rotation strings in simple xyz, ordered-axis,
  and compound (`add_rotation`) forms.
- PyVista placement now respects `g4placement_type`: volumes with `'passive'` mode apply the transpose
  of the rotation matrix so that world-frame positions are rendered correctly.
- Aligned ASCII output entries in `gvolume.py` and `gmaterial.py` for improved readability.

<br/>

## SQLite utilities

- Added `gemc-sqlite -n <filename>` to create a new empty SQLite database with the full GEMC geometry and materials schema.
- Registered `gemc-sqlite` as a console-script entry point in `pyproject.toml`.
- Fixed an uninitialized `experiment_filter` variable in `gsqlite.py` that would raise a `NameError` when `-ef` was not supplied.

<br/>

## GConfiguration

- `GConfiguration` now tracks all variations used during a run in a `self.variations` list.
- `show()` prints all variations instead of only the last one; the label switches from `Variation` to `Variations` 
   when more than one was used.
